# Repo Front Door Truth Pass

Use this checklist before treating the repo front door as publication-ready.

## Identity and classification

- [ ] `README.md` classifies the repo in the first screenful.
- [ ] `README.md` uses one canonical identity and does not compete with deeper docs.
- [ ] `REPO_INDEX.md` points to the real surfaces instead of duplicating them.

## What is real today

- [ ] `README.md` states what is shipped now.
- [ ] `README.md` states what the repo does not do.
- [ ] Entry points exist for Claude Code, skills, extension runtime, proof, and thesis/contract inspection.

## Claims boundary

- [ ] Local-first/no-auth/no-telemetry language matches repo truth.
- [ ] License wording matches the actual `LICENSE` and package metadata.
- [ ] Publication status and optional proof supplements are stated truthfully.

## Pass condition

Pass when a new reviewer can classify the repo, choose an entry path, and understand the repo''s current publication status in under 20 seconds.

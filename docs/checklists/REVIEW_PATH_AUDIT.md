# Review Path Audit

Use this checklist to confirm a public reviewer can inspect the repo in about three minutes.

## Path 1 - classify the object

- [ ] Start at `README.md`.
- [ ] Confirm `REPO_INDEX.md` and `docs/INDEX.md` are visible next steps.
- [ ] Confirm the repo surface count and public identity are clear.

## Path 2 - inspect doctrine and claims

- [ ] Open `TEAM_CHARTER.md` for doctrine.
- [ ] Open `docs/specs/PUBLIC_THESIS_OBJECT.md` for public-surface and scope truth.
- [ ] Confirm claims boundary language matches extension/runtime docs.

## Path 3 - inspect proof and release readiness

- [ ] Open `docs/indexes/PROOF_INDEX.md`.
- [ ] Open `proof/PROOF_MAP.md`, `proof/RELEASE_CHECKLIST.md`, and `proof/CAPTURE_RUNBOOK.md`.
- [ ] Confirm the proof lane does not overclaim screenshots when they are absent and labels `CAPTURE_RUNBOOK.md` as optional.

## Path 4 - inspect runtime surfaces

- [ ] Open `docs/indexes/EXTENSION_INDEX.md` for the VS Code extension surface.
- [ ] Open `docs/indexes/CLAUDE_COMPAT_INDEX.md` for the Claude compatibility path.
- [ ] Open `docs/indexes/SKILLS_INDEX.md` for the portable skills/resource-pack surface.

## Pass condition

Pass when each path works in a few clicks, uses current names, and does not rely on internal context or stale branch framing.
